🔍 prompts.py 调用机制详解

理解 prompts.py 何时被调用以及如何被调用

📌 核心答案

prompts.py 不会自动执行，它是一个模块定义文件，需要在其他代码中导入和调用。

Schema 参数可以通过两种方式指定：

  1. 在问题数据文件中指定（JSON 格式的 "kind" 或 "schema" 字段）
  2. 在代码调用时手动指定（作为函数参数传入）

🔄 完整调用流程

步骤 1: 问题数据输入

问题数据可以来自 JSON 文件或直接传入的字符串

{ "text": "中芯国际在晶圆制造行业中的地位如何？", "kind": "string" // 指定问题类型 }

步骤 2: QuestionsProcessor 处理问题

questions_processing.py 中的 _process_single_question() 方法读取 schema 参数

// questions_processing.py def _process_single_question(self, question_data: dict): question_text = question_data.get("text") schema = question_data.get("kind") // 获取 schema 类型 answer_dict = self.process_question(question_text, schema)

步骤 3: APIProcessor 构建提示词

api_requests.py 中的 get_answer_from_rag_context() 方法被调用

// api_requests.py def get_answer_from_rag_context(self, question, rag_context, schema, model): system_prompt, response_format, user_prompt = self._build_rag_context_prompts(schema) // ↑ 这里调用 _build_rag_context_prompts，传入 schema 参数

步骤 4: 根据 Schema 选择对应的 Prompt

_build_rag_context_prompts() 方法根据 schema 参数选择对应的 prompts 类

// api_requests.py def _build_rag_context_prompts(self, schema): if schema == "name": system_prompt = prompts.AnswerWithRAGContextNamePrompt.system_prompt response_format = prompts.AnswerWithRAGContextNamePrompt.AnswerSchema elif schema == "number": system_prompt = prompts.AnswerWithRAGContextNumberPrompt.system_prompt response_format = prompts.AnswerWithRAGContextNumberPrompt.AnswerSchema elif schema == "string": system_prompt = prompts.AnswerWithRAGContextStringPrompt.system_prompt response_format = prompts.AnswerWithRAGContextStringPrompt.AnswerSchema // ... 其他类型 return system_prompt, response_format, user_prompt

步骤 5: 使用 Prompt 调用 LLM

将构建好的 prompt 发送给 LLM API，获取结构化答案

answer_dict = self.processor.send_message( system_content=system_prompt, // 来自 prompts.py human_content=user_prompt.format(context=rag_context, question=question), response_format=response_format // 来自 prompts.py )

📝 Schema 参数指定方式

方式 1: 在问题数据文件中指定

JSON 文件格式（新版本）

[ { "text": "中芯国际的CEO是谁？", "kind": "name" // 指定为姓名类型 }, { "text": "中芯国际2024年营收是多少？", "kind": "number" // 指定为数字类型 }, { "text": "中芯国际是否宣布了分红政策变更？", "kind": "boolean" // 指定为布尔类型 } ]

处理方式：

// questions_processing.py if self.new_challenge_pipeline: question_text = question_data.get("text") schema = question_data.get("kind") // 从 "kind" 字段读取 else: question_text = question_data.get("question") schema = question_data.get("schema") // 从 "schema" 字段读取

方式 2: 在代码调用时手动指定

通过 Pipeline 调用

// pipeline.py def answer_single_question(self, question: str, kind: str = "string"): // kind 参数可以手动指定 answer = processor.process_single_question(question, kind=kind) return answer // 使用示例 pipeline.answer_single_question( question="中芯国际的CEO是谁？", kind="name" // 手动指定类型 )

📊 Schema 类型与 Prompt 类映射表

Schema 值 对应的 Prompt 类 适用场景 输出类型
"name" AnswerWithRAGContextNamePrompt 姓名类问题（如 CEO、高管姓名） str 或 "N/A"
"number" AnswerWithRAGContextNumberPrompt 数字类问题（如营收、资产等数值） float/int 或 "N/A"
"boolean" AnswerWithRAGContextBooleanPrompt 布尔类问题（是/否类型） bool (True/False)
"names" AnswerWithRAGContextNamesPrompt 名单类问题（高管列表、产品列表） List[str] 或 "N/A"
"string" AnswerWithRAGContextStringPrompt 文本类问题（总结、描述） str
"comparative" ComparativeAnswerPrompt 比较类问题（多公司比较） str 或 "N/A"

📍 prompts.py 的主要调用位置

1. api_requests.py

  • _build_rag_context_prompts() 方法
    • 根据 schema 参数选择对应的 Prompt 类
    • 返回 system_prompt, response_format, user_prompt
  • get_rephrased_questions() 方法
    • 使用 RephrasedQuestionsPrompt 进行问题重写
  • get_answer_from_rag_context() 方法
    • 调用 _build_rag_context_prompts() 获取 prompt
    • 使用 prompt 调用 LLM API

2. reranking.py

  • __init__() 方法
    • 初始化时导入 RerankingPrompt 相关提示词
    • 导入 RetrievalRankingSingleBlock 和 RetrievalRankingMultipleBlocks Schema

⚠️ 重要说明

1. prompts.py 是模块定义，不是可执行脚本

prompts.py 文件只包含类定义和函数定义，不会自动执行。它需要在其他模块中通过 import prompts 导入后使用。

2. Schema 参数是调用时传入的，不是自动判断的

系统不会自动判断问题类型。Schema 参数必须：

  • 在问题数据文件中明确指定（"kind" 或 "schema" 字段）
  • 或者在代码调用时作为参数传入

3. 调用时机

prompts.py 中的类和方法会在以下时机被调用：

  • 初始化时：reranking.py 在初始化时导入 RerankingPrompt
  • 处理问题时：api_requests.py 在处理每个问题时调用对应的 Prompt 类
  • 问题重写时：处理比较类问题时调用 RephrasedQuestionsPrompt

4. 默认 Schema

如果不指定 schema 参数，某些方法会使用默认值：

  • process_single_question(question, kind="string") 默认使用 "string"
  • 建议始终明确指定 schema，以确保使用正确的 Prompt 和输出格式

📝 总结

回答您的问题：

  1. prompts.py 什么时候会被调用执行？
    prompts.py 不会自动执行，它是在其他模块需要时被导入和调用的。主要调用时机包括：
    • 处理问题时（通过 api_requests.py）
    • 初始化重排序模块时（通过 reranking.py）
  2. 是程序自动判断的还是在哪指定的？
    Schema 参数不是自动判断的，需要在以下位置指定：
    • 问题数据文件（JSON）：使用 "kind" 或 "schema" 字段
    • 代码调用时：作为函数参数传入（如 kind="name"）
