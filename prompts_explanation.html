📝 prompts.py 文件详解

RAG 系统提示词管理模块完整解析

🎯 文件的核心作用

prompts.py 是 RAG（检索增强生成）系统的提示词管理中心，它集中定义和管理所有与 LLM（大语言模型）交互所需的提示词模板和结构化输出 Schema。

这个文件的主要职责包括：

  • 统一管理提示词：将所有 Prompt 定义集中在一处，便于维护和修改
  • 定义输出结构：使用 Pydantic 定义结构化输出 Schema，确保 LLM 返回格式一致
  • 支持多种问答场景：针对不同类型的问题（姓名、数字、布尔值、列表、比较等）提供专门的提示词
  • 保证输出质量：通过详细的指令和示例，引导 LLM 生成高质量、结构化的答案

💡 为什么要写这个文件？

  1. 代码组织性

    将所有提示词集中管理，避免在代码中散落各处，提高代码可维护性。

  2. 输出标准化

    通过 Pydantic Schema 强制 LLM 输出标准化格式，便于后续处理和解析。

  3. 场景适配

    不同问题类型需要不同的处理策略（如数字需要精确匹配，姓名需要完全一致），通过专门的 Prompt 类实现精准控制。

  4. 易于迭代优化

    提示词需要不断优化，集中管理使得调整和测试更加方便。

  5. 可复用性

    统一的提示词构建函数和模板，可以在不同场景中复用。

🔧 核心函数

build_system_prompt()

这是构建系统提示词的核心函数，它负责将指令、Schema 和示例组合成完整的系统提示词。

def build_system_prompt(instruction: str="", example: str="", pydantic_schema: str="") -> str: delimiter = "\n\n---\n\n" schema = f"你的回答必须是JSON，并严格遵循如下Schema，字段顺序需保持一致：\n```\n{pydantic_schema}\n```" # 组合指令、Schema 和示例 system_prompt = instruction.strip() + schema + example return system_prompt

功能说明：

  • 接收三个参数：指令文本、示例文本、Pydantic Schema
  • 使用分隔符 --- 清晰分隔不同部分
  • 确保输出格式为 JSON，并严格遵循定义的 Schema

📚 主要提示词类详解

1. RephrasedQuestionsPrompt

用途：将比较类问题拆解为针对每个公司的独立问题

示例："2022年哪家公司营收更高，'苹果'还是'微软'？"

输出：两个独立问题，分别询问苹果和微软的营收

2. AnswerWithRAGContextNamePrompt

用途：回答姓名类问题（如 CEO、高管姓名）

特点：要求姓名必须与问题中完全一致，且为全名

输出格式：包含分步分析、推理总结、相关页面、最终答案

3. AnswerWithRAGContextNumberPrompt

用途：回答数字类问题（如营收、资产等数值指标）

特点：严格的指标匹配要求，处理单位转换、负数、币种等

特殊处理：千、百万等单位转换，负数识别，币种校验

4. AnswerWithRAGContextBooleanPrompt

用途：回答布尔类问题（是/否类型）

特点：返回 True 或 False，基于上下文精确判断

5. AnswerWithRAGContextNamesPrompt

用途：回答名单类问题（如高管列表、产品列表）

特点：返回字符串列表，每个条目需与上下文完全一致

6. AnswerWithRAGContextStringPrompt

用途：回答文本类问题（如总结、描述）

特点：返回完整、连贯的文本段落

7. ComparativeAnswerPrompt

用途：基于各公司独立答案，生成比较问题的最终结论

特点：只基于已给出的答案，不引入外部知识

8. RerankingPrompt

用途：对检索到的文本块进行相关性评分和重排序

特点：评分范围 0-1，步长 0.1，用于提升检索质量

✨ 关键特性

结构化输出 Pydantic Schema 多场景支持 严格校验 分步推理 页面引用 错误处理 可扩展性

🎬 典型使用场景

场景 1：单公司问题回答

  1. 用户提问："'万科企业股份有限公司'的CEO是谁？"
  2. 系统检索相关文档
  3. 使用 AnswerWithRAGContextNamePrompt 构建提示词
  4. LLM 生成结构化答案，包含 CEO 姓名和相关页面

场景 2：比较类问题回答

  1. 用户提问："'苹果'和'微软'2022年哪家营收更高？"
  2. 使用 RephrasedQuestionsPrompt 拆解为两个独立问题
  3. 分别检索和回答每个公司的问题
  4. 使用 ComparativeAnswerPrompt 生成最终比较结论

场景 3：数值问题回答

  1. 用户提问："'万科企业股份有限公司'2022年总资产是多少？"
  2. 使用 AnswerWithRAGContextNumberPrompt 构建提示词
  3. LLM 严格匹配指标定义，处理单位转换
  4. 返回精确数值和推理过程

📝 总结

prompts.py 是 RAG 系统中的关键组件，它通过：

  • ✅ 集中管理所有提示词模板，提高代码可维护性
  • ✅ 标准化输出，确保 LLM 返回结构化的、可解析的 JSON 格式
  • ✅ 场景适配，针对不同问题类型提供专门的提示词策略
  • ✅ 质量控制，通过详细的指令和示例引导 LLM 生成高质量答案
  • ✅ 易于扩展，新增问题类型只需添加新的 Prompt 类

核心价值：将复杂的提示词工程系统化、模块化，使得 RAG 系统能够可靠地处理各种类型的企业年报问答任务。
